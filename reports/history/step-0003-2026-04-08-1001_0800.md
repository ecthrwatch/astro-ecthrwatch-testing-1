# Step 3: Refactor to domain-based i18n

**Commit:** `[step-3] Refactor to domain-based i18n — each language served at root /` — 2026-04-08 10:01:12 +0800

## What was done
- Removed path-based i18n routing (`/en/`, `/fr/`, `/zh-cn/` prefixes)
- Rewrote middleware to resolve locale from port (dev) or `LOCALE` env var (build)
- Single `src/pages/index.astro` renders based on `Astro.locals.locale`
- Added per-language build scripts: `build:en`, `build:fr`, `build:zh`
- Added `src/env.d.ts` for typed `Astro.locals`
- Deleted locale-prefixed page directories

## Issues
- Fundamental architecture change from path-based to domain-based i18n
- Each language now behaves as its own standalone site at root `/`
