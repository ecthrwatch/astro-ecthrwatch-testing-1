## What was done
- Installed `@astrojs/mdx` integration via `npx astro add mdx`
- Installed `js-yaml` for YAML parsing
- Copied 12 YAML data files from Hugo (`DONT_DELETE_common_files/theme_eureka_1/data/`) into `src/data/`:
  - `dictionaries/{en,fr,zh}/email.yaml` — email header labels
  - `dictionaries/{en,fr,zh}/original.yaml` — document link prefixes
  - `persons/{en,fr,zh}/person.yaml` — 157/83/67 entries with markdown links
  - `signatures/{en,fr,zh}/signature-1-default.yaml` — contact blocks
- Created `src/data/loader.ts`:
  - `getDict()`, `getPerson()`, `getOriginalText()`, `getSignature()`
  - `renderInlineMarkdown()` — converts `[text](url)` to `<a>` tags
  - `localeToDir` mapping: our `zh-cn` to Hugo's `zh`
  - File-level caching with `Map`

## Issues
- Person YAML values contain markdown links — critical to handle correctly. The `renderInlineMarkdown()` regex handles the `[text](url)` pattern which is the only markdown used in these files.
